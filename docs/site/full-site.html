How it works

Give your agent the Kiln skill definition. It handles the rest.

Install & discover

One command installs Kiln. kiln setup auto-discovers printers on your network and saves config.

Agent reads the skill

Your AI agent reads SKILL.md ‚Äî a structured document defining every command, safety level, and workflow pattern.

Safely operate hardware

The agent slices, uploads, prints, and monitors ‚Äî with Kiln enforcing temperature limits, G-code validation, and pre-flight checks at every step.

Human stays in control

Irreversible actions (start print, cancel, raw G-code) require human confirmation. The agent asks, the human approves.

Built for agents, works for humans

Every command returns structured JSON. Agents parse it, humans read it.

# Check what the printer is doing
$ kiln status --json
{"status": "printing", "progress": 67.2, "temps": {"tool": 210, "bed": 60}}

# Slice an STL and start printing (with safety checks)
$ kiln slice model.stl --print-after --json
{"status": "success", "gcode": "model.gcode", "print_started": true}

# No local printer? Get a manufacturing quote
$ kiln order quote model.stl -m pla_standard --json
{"price_usd": 12.50, "lead_time_days": 5, "quote_id": "q_abc123"}

# Compare local vs. outsourced cost
$ kiln compare-cost model.gcode --fulfillment-material pla_standard --json
{"local_cost_usd": 1.20, "fulfillment_cost_usd": 12.50}
